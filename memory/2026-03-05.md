## Slug Query Breakthrough (18:47 UTC)

**F1 Master's hint:** Use slug parameter instead of condition_id polling

**Pattern discovered:**
- URL: `https://polymarket.com/event/btc-updown-5m-1772736300`
- API: `https://gamma-api.polymarket.com/markets?slug=btc-updown-5m-1772736300`
- This returns complete market data + token IDs

**Verified with live market 1772736300:**
- ✅ Market metadata retrieved
- ✅ YES/NO prices: $0.505 / $0.495
- ✅ CLOB token IDs: 2 tokens found
- ✅ Order book: 61 bids, 38 asks
- ✅ All data accessible

**Simplified Option B approach:**
1. Calculate next expected slug: `btc-updown-5m-{base_id + 300}`
2. Query by slug (every 5 min)
3. Extract token_ids → subscribe to CLOB
4. Collect prices + order books every 10s
5. Track until market resolves

**Key insight:** Much simpler than condition_id polling. Directly uses URL slug format.
