## Option B Deployment Deep Dive (03:49–03:59 UTC)

**Critical Issue Identified:**
- V1/V2 autonomous collection ran 7.5 hours, captured only 5 resolutions (expected ~90)
- Order book collection: 0 records (file never created)
- Root cause: **Hardcoded market base ID (1772738400) only checked 5 markets, all expired after 2.5 hours**

**Root Cause Analysis (Opus):**
1. **Market ID scheme misunderstood** — Not hardcoded. Formula: `(unix_time // 300) * 300`
2. **No sliding window** — Script checked same 5 IDs forever instead of discovering new markets
3. **Order book only from active markets** — Script was tracking only closed markets (no data)
4. **Markets close with 5-10 min delay** — Must poll past-closed markets for 15 minutes
5. **Zero error logging** — `except: pass` meant no visibility into what broke

**V3 Fixes Applied:**
1. ✅ Dynamic market ID: `current_market_id = (unix_time // 300) * 300`
2. ✅ Sliding window: Tracks current market ± 3 offsets (6 markets total)
3. ✅ Order books from ACTIVE markets only: `active=true AND closed=false`
4. ✅ Proper error logging: Every failure logged for diagnosis
5. ✅ Separate resolution tracking: Independent from closure detection
